# Deep Audit: PocketAgent vs z.ai/Kimi/Claude Code

## Key Findings from Anthropic's "Writing Effective Tools for Agents"

### 1. FEWER TOOLS, RICHER DESCRIPTIONS
"More tools don't always lead to better outcomes."
"Tools can consolidate functionality, handling potentially multiple discrete operations under the hood."

**Current PocketAgent:** 9 separate tools (bash, file_read, file_write, file_list, str_replace, grep, glob, web_fetch, web_search)
**z.ai/Kimi approach:** Fewer high-level tools with rich descriptions

**Action:** Keep all 9 tools but make the system prompt guide the AI to use them efficiently. The tools are fine — the problem is the AI doesn't know WHEN to use which tool.

### 2. TOKEN EFFICIENCY
"Optimizing the quantity of context returned back to agents in tool responses."
"Restrict tool responses to 25,000 tokens by default."

**Current PocketAgent:** Tool results can be up to 256KB — way too much context.
**Action:** Reduce max output to 30KB, add truncation messages.

### 3. MEANINGFUL ERROR RESPONSES
"Prompt-engineer your error responses to clearly communicate specific and actionable improvements."

**Current PocketAgent:** Errors are raw exception messages.
**Action:** Make errors actionable — tell the AI exactly what to do to fix it.

### 4. UI: COLLAPSE TOOL CALLS
From Reddit: "Claude Code should collapse the tool calls and thinking"
"Claude often collapses its tool calls as it does its task/thinking but you can click on it later"

**Current PocketAgent:** Tool calls auto-expand while running but stay visible.
**Action:** Auto-collapse tool calls after completion. Show minimal status (✓/✗ + summary).
